# 参考搜索与证据集成

> 外部搜索是增强，不是依赖。优先使用当前环境实际提供的浏览、搜索、连接器或已登记资料库；全部不可用时，核心设计与审查流程仍应完成。

## 1. 能力探测

开始前只检查与任务相关的能力：

1. 能否搜索公开网页或官方文档？
2. 能否读取用户提供的 URL、截图或文件？
3. 是否有用户已登记且授权访问的灵感源？
4. 是否能保存必要的参考图到项目生成目录？

不要假设某个 MCP、浏览器、函数名或登录态一定存在。能力缺失时直接降级，不要求用户安装第三方工具才能获得基本结果。

## 2. 来源优先级

1. 用户提供的品牌规范、项目文件和真实资产。
2. 品牌官网、产品文档、平台设计指南等一手来源。
3. 可验证的真实产品页面和截图。
4. 高质量二手分析，仅用于补充解释。

涉及当前产品状态、版本、规格或法规时必须实时验证。事实搜索与视觉灵感搜索要分开：前者回答“是否真实”，后者回答“如何表达”。

## 3. 搜索策略

针对同一问题使用至少两个不同角度，例如：

- 屏幕类型：`onboarding empty state mobile`
- 任务模式：`bulk edit confirmation enterprise`
- 同类产品：`company product pricing page`
- 设计原则：`accessible data table keyboard pattern`

深度研究通常需要 3 个以上独立查询；快速参考可少于此数。控制结果数量，只保留真正支持判断的证据。

## 4. 证据门槛

每条引用至少记录：来源、访问日期、可观察事实、与建议的关系。截图还需验证其确实包含声称的模式，不凭缩略图标题推断。

引用证据时区分：

- **事实**：来源直接陈述或页面直接呈现。
- **推断**：由多个事实组合得出的判断，应明确标注。
- **灵感**：可迁移的视觉或交互模式，不代表行业标准。

不同来源冲突时，优先一手来源与当前版本；无法消解时报告冲突，不强行选边。

## 5. 下载与存储

仅在交付确实需要时下载截图。默认保存到 `.monkren/references/{topic}-{date}/`，并确保 `.monkren/` 已被忽略。

- 文件名使用可读 slug，不保存 token、cookie 或认证信息。
- 记录原始 URL 和访问日期。
- 遵守来源许可，不批量复制受版权保护的作品。
- 报告使用相对路径，缺图时不留下伪造文件名。

## 6. 降级模式

| Mode | 可用能力 | 行为 |
|---|---|---|
| Full | 项目上下文 + 外部来源 | 用真实证据补强判断 |
| Partial | 只有其中一类 | 明确数据边界，继续完成 |
| Local | 无外部访问 | 基于项目文件与通用原则，不编造外部事实 |

降级提示保持简短，例如：“本次无法访问外部参考，结论仅基于仓库与通用设计标准。”不要让降级提示淹没报告。

## 7. 安全边界

- 登录外部服务、写入用户级配置或下载大量文件前需要明确授权。
- 不输出、记录或提交凭据。
- 不绕过付费墙、访问控制或网站限制。
- 用户只要求审查时，外部搜索不会扩大为修改项目或登记新来源的权限。
