Second Opinion · Cross-stage Advisor

先判断该不该做，再判断怎么把它做好。

第一性原理负责质疑需求、删除浪费和识别硬约束；产品品味负责守住用户承诺、端到端体验和真正值得做到极致的时刻。不是双人格表演，是一个可执行裁决。

01 · First Principles

它为什么必须存在？

逐项追问需求来源，区分硬约束与习惯，先删除，再简化；只对低风险、可逆决策快速试验。

02 · Product Taste

用户为什么会在乎？

把产品压缩成一句承诺，保护核心旅程和端到端体验，把质量集中在决定产品是否成立的时刻。

  1. 01安全与真实性
  2. 02核心用户价值
  3. 03技术可行性
  4. 04体验一致性
  5. 05速度与成本
> 让次设计 agent 判断这个方向该不该做 Read-only · One decision · Two dissents
Disciplines · 4 评分纪律

诚实底线。99% 是 checklist，monkren 是 judgment。

5 维度是评分对象，4 纪律是评分方式。装上后这 4 条自动启用——不需要手动配置。

01 禁止评分通胀全维度 ≥7 时强制自检
02 禁止平均上浮取最差持续段，非平均分
03 评分必须引证文件 + 行号 + 元素
04 创新性允许低分5/10 对生产交付物合理

Difference · 1 个 design agent，不是 1 个 agent

monkren = 1 个 design agent。通用 agent 是 1 个 coder。

它们回答"怎么做出来"。monkren 额外回答"做出来好不好 · 像不像你 · 怎么修到 9 分"。

01 维度方法论 通用 coding agent按 prompt 实现，能跑就行 AI 设计工具 · Review 工具按组件生成 / 按 checklist 打勾 monkren5 维度评审（哲学 / 层级 / 细节 / 功能 / 创新）
