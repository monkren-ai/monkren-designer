以下是 2026-07-15 审查时的 v6.0 基线快照，不代表当前 v6.1 状态。当时的 landing page 采用 Graphic Monochrome Canvas 2.0 哲学：纯黑/纯白/单色灰 + 0 圆角 UI（仅 .mark 50% 例外） + 1px 边框 + 3 字体职责分明 + DEPTH meter 滚动驱动视觉签名。审查当时发现 hero 只有 4 条 query、缺少 03 设计执行阶段，compat 数量不一致，并存在未 token 化数值；这些问题已在后续版本处理。该案例保留用于展示设计建议如何从审查基线推导方向。

Recommendation

→ Monochrome Hardening

当前设计已具 Graphic Monochrome Canvas 2.0 哲学骨架，Monochrome Hardening 方向改动最小、风险最低。3 个改动（DEPTH meter 升级 scrub / 3 case 卡片与 discipline 视觉语言统一 / --color-warn 收窄到 3 处）全部围绕"深化"展开，不引入新色彩、不增加字体、不破坏 1+3+1 结构。每个改动都基于 review-report.html 的 Fix 列表，与 Score Improvement 路径 1+2+4 完全对应。适合下一轮迭代直接执行，与现有 token 体系完全兼容。
