0. 一句话诊断

Monkren Design v4.0 是一个 10/10 的 design library，但正在被当作 product 卖。用户进来看到 9 个 section、80 张哲学卡、40 哲学库、6 平台、10 视角、3 案例——他不知道 monkren 是什么。

问题不是质量，是shape。Library 形状做不了 product。Library 让用户敬佩，product 让用户行动。

关键判断 Monkren 的核心资产是 5 维度评审 + 评分纪律 + 反 AI slop 黑名单 + 硬编码检测——4 个东西。其余 17 个 skill、80 种哲学、6 种平台、10 个视角、3 阶段方法、9 份 reference 都是好的，但不应该出现在用户面前。

1. 三层战略 · 砍 / 留 / 转

所有 21 个 skill + 9 份 reference + 80 张哲学卡 + landing page 9 个 section，按乔布斯三道过滤器分类。

Layer 1 · 砍

对用户隐藏。从产品表面消失。

这些东西好，但用户不需要看见。它们是 monkren 内部使用的工具，不是产品功能。

资产为什么砍保留方式
80 张哲学卡（landing page section 04） 用户来 Monkren 不是来学 Pentagram / Müller-Brockmann 的。这是教育内容，不是产品功能。 迁移到 references/philosophy-library.md，作为 agent 内部推荐知识库。landing page 不再展示。
17 流派分类 17 流派 = "我什么都知道" 信号，= 决策瘫痪。对一个 review tool 是反向价值。 压缩为 3 大类（克制/系统/反叛），agent 内部按 17 流派工作，对外只说 3 类。
8 个 create skill（wireframe / make-a-prototype / make-a-deck / make-tweakable / generate-variations / design-system-extract / component-extract / design-improve） 打不过 Bolt / v0 / Cursor 自己的创作能力。这是 monkren 做不好的领域。 全部标记为 deprecated。三个月后无用户使用则物理删除文件。
10 视角审查（perspectives.md） 5 维度已经够 review 90% 设计。10 视角是给设计系统极客看的。 合并到 5 维度评审的"补充"子维度。5 个主维度，3 个补充视角。
6 平台专项（platforms.md） monkren 自身只擅长 web landing page 审查（因为 v4.0 只做过这一种）。其他平台是空头支票。 保留文件但标"实验性"。先只做 web 一种平台，做透再说。
9 个 section 的 landing page 140KB / 3401 行的 landing page = 0% 转化。用户 7 秒内没看到 "我为什么要装"，就走了。 重写为 1 屏结构。详见 §3。
Layer 2 · 留

核心资产——这是产品的 1,000 songs。

这 4 个东西是 monkren 的 identity。砍掉它们，monkren 不存在。

资产为什么留展示方式
5 维度评审（哲学一致性 / 视觉层级 / 细节执行 / 功能性 / 创新性） 这是 monkren 独有的判断框架。市面所有 review tool 都只打分不解释维度。monkren 解释了。 每个审查报告都包含 SVG 雷达图。5 维度是 monkren 的"指纹"。
4 条评分纪律（禁止评分通胀 / 禁止平均上浮 / 评分必须引证 / 创新性允许低分） 这是 monkren 的 taste。99% 的 review tool 是 checklist，monkren 是 judgment。 每个评分后面强制跟引证（文件路径 / 行号 / 元素名）。这是 monkren 的诚实底线。
反 AI slop 9 类黑名单 这是 monkren 的 差异化赌注。未来 3 年，"独特"会是最稀缺货币。monkren 在赌这个。 作为独立命令：> 审查这个设计是不是 AI slop。这是 monkren 的招牌 query。
硬编码值检测（HEX/RGB/字体/4·8 倍间距） 这是 monkren 最可自动化的能力。无需 LLM 推理，纯规则扫描。给用户即时反馈。 作为独立命令：> 检测这段代码里的硬编码值。这个能 5 秒出结果。
Layer 3 · 转

不是删，是换一种方式存在。

这些是好东西，但需要换形态。位置不对、形式不对、用户接触路径不对。

资产问题转型方式
"双重身份"定位（审查员 + 创作者） hedging。用户记不住。同时做两件事通常意味着两件都做不好。 对外只说审查员。创作者身份退到内部 mention，6 个月后彻底砍掉。
5 阶段 skill 矩阵（Discover/Define/Create/Review/Deliver） 5 阶段是给 agent 用的路由表，不是给用户看的菜单。用户来 monkren 只想做一件事：审查。 5 阶段保留在 SKILL.md 内部。landing page 上只展示 1 个入口："审查"。
3 份 case study（Design Review / Score Improvement / Design Suggestion） 这是 最有说服力的资产，但放在 landing page 中段被淹没了。 搬到 landing page 第一屏下面。3 个 case 轮播，10 秒内让用户看到 "monkren 真的能做事"。
9 份 reference 9 份 reference 是给 agent 加载的，不应该出现在用户 landing page 上像"深度阅读"菜单。 landing page 隐藏。改为 GitHub 链接在 footer 出现一行："monkren 的 9 份 reference 在 GitHub"。
Graphic Monochrome Canvas 设计语言 方向对了，但表达太学术。Müller-Brockmann 致敬用户感受不到。 保留配色和字重，但增加 1 个 反 AI slop 的视觉签名——也许是默认 hover 时图片从灰阶变彩色的 micro-interaction 本身的设计哲学化。

Monkren：你的 AI 设计的 slop 检测器。

iPod = 1,000 songs in your pocket。Monkren = 1 个命令，告诉你的设计是不是 AI 生成的最大公约数。

  2. slop-check（招牌能力）
  3. hardcode-scan（招牌能力）

其他 17 个 skill 全部标 deprecated。6 个月后无用户使用则物理删除。

3. 新 landing page 规范

砍 80% 之后的 landing page 长什么样。

3.1 总体原则

  • 一屏定理：用户在 1440×900 视口下，hero 一屏内必须看到 "是什么 / 为什么装 / 怎么装" 三件事。
  • 5 秒测试：眯眼 5 秒，屏幕上仍能识别 hero 区块的就是合格设计。
  • 0 scroll 决策：用户无需滚动页面就能做 "装 / 不装" 决定。滚动只是为了取信。
  • 0 装饰：每个像素要么是 navigation，要么是 content，要么是 affordance。

3.2 页面结构（1 + 3 + 1）

位置区块内容存在理由
Hero（100svh） 品牌 + 1 句话 + 3 个 query + 1 行 install Monkren。
你的 AI 设计的 slop 检测器。

> 审查这个设计是不是 AI slop
> 扫描硬编码值
> 5 维度评审

npx skills add monkren-ai/monkren-designer 		7 秒内让用户做出 "装 / 不装" 决定。
Proof（1 屏） 3 份 case study 横向对比 Review / Improvement / Suggestion——3 张静态截图，10 秒内证明 monkren 真的能做事。 用户需要 "ok 看起来真的有用" 的证据。
Method（1 屏） 5 维度 + 4 条纪律（一句话版） 5 维度 = monkren 的指纹。
4 条纪律 = monkren 的诚实底线。
一行话讲完，不展开。 		深度用户需要知道 "怎么评分"。
Install（1 屏） 1 行 install + 兼容性列表 npx skills add monkren-ai/monkren-designer
Claude Code / Cursor / Codex / OpenClaw / Hermes		 消除 "怎么装" 的最后摩擦。
Footer（1 行） GitHub + License + 9 份 reference 链接 深度用户从这里去 GitHub 看 reference。 把 reference 链接从 hero 移到 footer。
总长度：5 屏之内。文件大小目标：< 20KB。行数目标：< 400 行。

3.3 视觉规范（Graphic Monochrome Canvas 2.0）

保留原配色（纯黑 / 纯白 / 单一灰）+ 字重（400 唯一）+ 几何（1px 边框、无圆角）。增加 1 个 micro-interaction 作为视觉签名：

  • Slop-meter：hero 区域右侧一个 1×1 的进度条，随页面 scroll 距离增长，从黑变白。让用户 感受到 "你正在输入的就是 slop 制造机"。

3.4 文案规范

  • Hero headline：≤ 8 个字。"提交设计" 不行；"Slop 检测" 行。
  • Subhead：≤ 14 个字。"30 秒告诉你是 AI 生成的最大公约数" 比 "5 维度评审 + 10 视角 + 40 哲学 + 17 流派 + 3 阶段" 好 100 倍。
  • 0 emoji：monkren 是 design tool，emoji 是 AI slop 重灾区。
  • 0 形容词：删除 "专业"、"insanely great"、"revolutionary"。用动词："审查 / 检测 / 评分 / 修复"。
