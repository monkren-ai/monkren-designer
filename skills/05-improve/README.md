# 05 · 设计改进（Design Improvement）

> monkren 设计生命周期的**第 5 阶段**。基于审查报告修复——把问题变成具体可执行的修复路径。

## 本阶段 skill

| Skill | 状态 | 用途 |
|-------|------|------|
| [design-improve](./design-improve/) | ✅ 活跃 | 基于 5 维度评审报告生成修复路径 |

## 阶段问什么

- **怎么修**？——把审查报告的 P0/P1 项变成具体动作
- **修完怎么验证**？——回到 04 设计审查

## 核心方法

- 修复必须**引用审查报告的具体条目**——不是"重新设计"，而是"把 hero 文案改为 48pt，引用审查报告 §3.2"
- 修复优先级按严重度：P0 blocker → P1 high → P2 medium
- 修完后回到 04 审查验证

## 阶段产物

- 修复后的设计（或修复路径文件）
- 修复清单：每条修复对应 1 个审查报告条目
- 自评：修完后再做一次 5 维度评审，对比改进前后

## 反馈循环

- 修完后 → 回到 **04 设计审查**验证
- 修完后产生新需求 → 回到 **01 调研定义**（比如要做一个新页面）
- 修完后方向不通过 → 回到 **02 创作定义**（重新选方向）
