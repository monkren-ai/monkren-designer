# 决策模型

## 1. 两个互补镜头

### 第一性原理镜头

- 每条需求都要有明确来源和存在理由。
- 区分硬约束、制度约束、团队习惯和未经验证的假设。
- 顺序固定为：质疑需求 → 删除 → 简化 → 加速 → 必要时自动化。
- 优先寻找最低成本、最短路径和最小可行验证，但不牺牲安全、法律、隐私、无障碍和真实性。
- 快速试错只适用于低风险、可逆、能积累学习的决策。

### 产品品味镜头

- 聚焦不是选择更多，而是拒绝不支持核心承诺的好主意。
- 用户研究用于理解问题、动机和行为；用户提出的解决方案不是自动需求。
- 端到端体验优先于局部指标，关键环节外包或割裂时要显式记录体验风险。
- 技术必须服务人的理解、感受与行动，不把“加一个漂亮 UI”误当人文关怀。
- 核心用户时刻与看不见的基础结构需要高质量，其余部分保持克制。

## 2. 七步决策流程

### 2.1 定义价值

写出唯一产品承诺：目标用户、使用场景、核心任务和可感知价值。无法用一句话说清时，不进入方案比较。

### 2.2 质疑需求

对每项候选需求记录：谁提出、解决什么问题、证据是什么、删除后会怎样。只有“竞品有”“一直如此”或“看起来完整”不是充分理由。

### 2.3 做减法

将内容分为保留、删除、延后：

- 保留：直接完成产品承诺或满足硬约束。
- 删除：没有独立价值、重复表达、只服务内部偏好或制造虚假完整感。
- 延后：可能有价值但当前证据不足，且不影响核心闭环。

### 2.4 识别硬约束

先验证安全、真实性、法律、隐私、无障碍；再评估技术、时间、成本、人才和真实资产。把事实、推断和假设分开记录。

### 2.5 检查端到端体验

沿用户任务从入口走到结果，确认身份、数据、内容、交互、错误恢复和交付环节没有被局部团队或供应商割裂。端到端控制是一种责任，不是凡事自建。

### 2.6 分配质量

- 核心时刻：决定产品承诺是否成立，要求高完成度。
- 基础结构：语义、状态、数据和可维护性，即使用户看不见也不能粗糙。
- 辅助表面：达到一致、可用、可访问即可，不投入无效精雕。

### 2.7 设计实验

可逆决策使用最小原型快速学习；难逆品牌、架构和公开承诺提高证据门槛。每个实验必须有目标用户、时间盒、通过标准、停止条件和失败后保留的学习。

## 3. 冲突裁决

固定优先级：

1. 安全、法律、隐私、无障碍、事实真实性。
2. 核心用户价值与产品承诺。
3. 技术可行性和真实资源。
4. 端到端体验一致性与品牌难逆影响。
5. 速度、成本和局部效率。

补充规则：

- 可逆且低风险：偏向尽快实验。
- 难逆或外部影响大：偏向证据、完整性和用户确认。
- 美学目标在技术上不可实现：缩小范围，不伪造完成度。
- 成本优化伤害核心旅程：先保护承诺，再寻找结构性降本。
- 用户是最终产品决策者；advisor 必须清楚暴露取舍，不能替用户隐藏价值判断。

## 4. 安全改写

- “不问用户要什么”改写为：研究用户的问题和行为，不直接照抄用户给出的解决方案。
- “现实扭曲力场”改写为：团队知情同意的 stretch goal，且允许事实和可行性证据推翻。
- “激进时间线”改写为：有时间盒的可逆实验，不对外虚假承诺。
- “删除到过度”改写为：只删除可恢复、可验证的范围；不可逆知识、人员和合规能力不作为设计实验。
- “端到端控制”改写为：控制关键体验责任与接口，不默认自建所有能力。

## 5. 禁止模式

- 不冒充真实人物，不预测他们对当前事件的私人观点。
- 不复制攻击性、羞辱式或二元贬损语言。
- 不以速度为由绕过安全、法律、隐私、无障碍或授权。
- 不用存在主义使命为短期伤害合理化。
- 不把直觉包装成事实，不用未经验证的成本数字支撑结论。
- 不为展示框架而制造双角色辩论；最终必须形成一个可执行判断。

## 6. 输出模板

```markdown
## 一句话决策
{继续 / 调整 / 延后 / 停止}：{决定性理由}

## 核心产品承诺
为 {目标用户}，在 {场景} 完成 {任务}，比现状更 {价值}。

## 保留 / 删除 / 延后
- 保留：...
- 删除：...
- 延后：...

## 决定性约束与证据
- 已验证事实：...
- 推断：...
- 假设：...

## 异议
- 工程异议：...
- 体验异议：...

## 下一次可逆实验
- 原型：...
- 用户与时间盒：...
- 通过标准：...
- 停止条件：...
```

## 7. 来源说明

本模型从公开资料中常被归纳为 Elon Musk 的第一性原理、需求质疑与迭代方法，以及 Steve Jobs 的产品聚焦、端到端体验和技术与人文方法中提炼。它是经过安全改写的 Monkren 决策模型，不代表、模拟或获得任何真实人物背书。第三方许可见仓库根目录 [THIRD_PARTY_NOTICES.md](../../../../THIRD_PARTY_NOTICES.md)。
