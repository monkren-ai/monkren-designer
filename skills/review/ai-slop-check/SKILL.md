# AI Slop Check：检测并修复通用的 AI 美学痕迹

审查当前设计中那些传递"AI 生成模板"信号的视觉套路，并修复发现的痕迹。这些模式读起来像是默认产物，而非有意为之——一个看起来像上百个其他 AI 输出的设计，无法成为用户自己的设计。

每条规则都**以正向为先**：先给出应追求的默认，再给出需要检测和替换的模式。在写作时，倾向于遵循默认；在审查时，扫描检测模式。

## Phase 1：确定审查范围

审查用户刚刚编辑或询问的 HTML/CSS 文件；否则审查本次会话中修改过的文件；如不明确，则询问。阅读文件并浏览引用的 CSS、token 和组件文件，以便能解析出实际值。

## Phase 2：单次审查

在单次审查中应用下面的每条规则——这些模式足够明显，并行分发属于过度设计。报告每一次检测，包括不确定或低严重度的情况，并给出置信度和严重度估计。覆盖是此阶段的工作；过滤在 Phase 3 或 `polish-pass` 聚合时进行。

### 1. 渐变——纯色或微妙、同色调

**默认：** design system 的纯色，或微妙的同色调双停点渐变。纯色几乎总是更强。

**检测并替换：** 彩虹 / 3+ 色渐变；在 hero、按钮或大块面上使用饱和的紫到粉、橙到粉或其他"潮流"混合；在不改善可读性或层级的图像上叠加渐变。

### 2. emoji——仅限功能性或 brand 驱动

**默认：** 不用 emoji，除非 brand 在现有材料中使用、emoji 是功能性的（状态指示、分类标记），或用户要求。

**检测并移除：** 在标题、按钮或列表项前加 emoji（`🚀 Get Started`）；重复的 emoji 填充（`🎉🎉🎉`）；无意义的 emoji 项目符号。如果布局依赖 emoji 的视觉重量，改用真正的图标（Feather、Material、Phosphor、Heroicons）或更好的排版层级。

### 3. 卡片——阴影、细边框或背景分隔

**默认：** 微妙的阴影、细的全周边框，或背景分隔。将 `border-left: 4px solid` 留给语义强调（callout、alert、状态）。

**检测并替换：** `border-radius: 12px` + `border-left: 4px solid` 作为*默认*卡片样式——这个组合读起来像"默认 SaaS 模板"。仅当左侧边框承载该语义含义，或来自你要匹配的现有 design system 时，才保留它。

### 4. 图像——真实、有授权，或诚实的 placeholder

**默认，按优先级：** 真实摄影（Unsplash、brand 资产）；专业插画；诚实的 placeholder——条纹背景配等宽标签，如 `product shot (1200×800)`。一个 placeholder 胜过一张糟糕的插画：它表明"需要素材"，而不是假装是最终成品。

**检测并替换：** 非熟练插画师绘制的人物、场景或概念 SVG 插画；"AI 风格"角色艺术（巨大的头、纯色色块、千篇一律的姿势）；把 placeholder 质量的装饰当作最终成品呈现。

### 5. 字体——有意为之的选择

**默认：** 与 brand 调性或媒介匹配的字体。在没有 brand 的情况下，建议 2–3 个匹配设计调性的备选（几何风、人文风、现代风、古典风），让用户挑选。

**检测并质疑** 对 Inter、Roboto、Arial、Fraunces 或纯系统字体栈的默认使用。仅当 brand 指定或用户确认时才保留。不要默默地把一种通用字体换成另一种通用字体。

### 6. 色彩——带微妙色调的白色与黑色

**默认：** 白色和黑色按调色板调色——暖色 `#FFFAF0`/`#2D2118`、冷色 `#F5F7FA`/`#1F2937`、中性 `#FAFAFA`/`#1A1A1A`。

**检测并替换：** 精确的 `#FFFFFF` 背景配精确的 `#000000` 文字——刺眼、冷漠、未完成。

### 7. 色彩值——追溯到 token 或和谐调色板

**默认：** 每个颜色都追溯到 design token、brand 变量或 `oklch()` 派生的调色板（从零构建时用 `oklch()` 保持亮度和色度一致）。

**检测并整合：** 无处追溯的颜色——同一个文件里出现五种略有不同的蓝色，意味着颜色是内联编造的。

### 8. 间距——对齐到 4px 或 8px 刻度

**默认：** 间距 token（`--space-xs: 4px` 到 `--space-2xl: 64px`）；4 或 8 的倍数显得有意为之。

**检测并替换：** 偏离刻度的值（`padding: 7px 15px`、`margin: 18px`、`gap: 13px`）——它们显得混乱。

### 9. editorial-warm 住宅风格——要么有意，要么不存在

**默认：** 为 brief 选定的美学方向（参见 `frontend-aesthetic-direction`）。这种暖色编辑风对编辑、酒店和作品集工作而言是正当的——前提是它追溯到 brand 或一个明确承诺的方向。

**检测并质疑** 在没有 brand 理由的情况下出现该组合：`#F4F1EA` 系列的奶油色 / 暖白背景；作为默认字体的衬线展示字（Georgia、Playfair Display、Fraunces）；标题中的斜体强调词；赤陶 / 琥珀色强调色。其中任何一项都可以是有意的。但全部凑在一起——尤其是在 dashboard、开发工具、fintech、医疗或企业级界面上——就是默认模板的样子，相当于今天的紫色渐变。用已承诺的方向替换，或在不存在时标记给用户。

## Phase 3：修复并总结

直接应用修复。当多个选项都合理时（例如选哪种非 Inter 字体），挑一个最站得住脚的默认值并记录，以便用户能覆盖。总结：按类别分类发现的套路、应用的修复、留给用户的开放问题。
