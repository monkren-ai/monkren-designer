# 层级与节奏审查

审查当前设计的视觉层级与节奏——这两项品质最能区分"有意为之"与"AI 生成"——并修复发现的问题。**层级引导视线**（什么先被看到、第二、第三）；**节奏让设计显得有意**（重复中带有策略性变化）。当它们到位时，设计读起来毫不费力。

## Phase 1：确定审查范围

审查用户刚刚编辑或询问的 HTML/CSS 文件；否则审查最近修改的设计文件；如不明确，则询问。阅读文件及其引用的样式，并记录媒介（slide / 页面 / 移动端 / dashboard）——规则因上下文而异。

## Phase 2：并行启动两个审查 agent

使用 ${AGENT_TOOL_NAME} 工具在一条消息中并发启动两个 agent。

明确指示两个 agent：报告发现的每一个问题，包括不确定和低严重度的情况，并给出置信度和严重度估计。覆盖是 agent 的职责；过滤在聚合阶段（Phase 3）进行。

### Agent 1：层级审查

对每个屏幕、slide 或主要区块：

1. **primary / secondary / tertiary。** 什么应该先被看到、第二、第三？如果你说不清，层级就是坏的。
2. **大小。** 标题明显比正文大；primary CTA 比次要操作大。标记"相似内容用差异极大的尺寸"（不一致）和"不同重要性的内容用相似尺寸"（扁平）。
3. **色彩。** primary 操作用饱和的 brand 色，secondary 用中性色，弱化的用浅灰。标记"所有元素同色"（无信号）和"最不重要的元素最亮"（信号错误）。
4. **字重。** 标题加粗、正文常规、说明文字用 light。标记"全加粗"和"全常规"。
5. **位置。** 视线从左上开始（在 LTR 中）；最重要的内容应在黄金位置。标记被埋在右下角的 primary 元素。
6. **密度。** 宽松的间距传达"注意这里"，紧凑的传达"辅助信息"。标记"重要内容被挤在一起而填充内容却舒展"——那是反过来的。
7. **5 秒测试。** 一个首次用户应该在 5 秒内知道该看什么、该做什么。

### Agent 2：节奏审查

对整个文档：

1. **间距刻度。** 所有 padding/margin/gap 值对齐到一致的刻度（4px 或 8px 的倍数）。列出隐式使用的刻度并标记离群值（`padding: 7px`、`gap: 13px`）。
2. **字体刻度。** 标记任意尺寸（在一个 16/20/24 的设计中出现 `17px`、`23px`）。
3. **重复。** 网格中的卡片、列表项、功能块共享 padding、gap、尺寸、结构。标记那些微妙差异的近似重复——要么完全相同，要么有意不同，不要中间状态。
4. **策略性变化。** 长页面或 deck 偶尔打破模式（背景切换、更宽的区块、居中的 CTA）。标记完全统一（单调）和每个区块都不同（混乱）。
5. **色板纪律。** 3–5 种颜色加上 tints/shades。标记 8+ 种独立颜色，或在不同位置出现略微不同的蓝/灰。
6. **区块结构。** 区块在视觉上可区分（背景变化、分割线、padding 切换），但要一致。标记"无分隔"和"分隔样式过多"。
7. **对齐。** 标记偏离网格几像素的元素——是不一致的 margin，而非有意的偏移。

## Phase 3：聚合并修复

等待两个 agent，聚合发现，然后直接修复：

- 随意的间距 / 字体大小 → 对齐到文件的刻度（缺失时定义一个——4px 或 8px 的倍数）
- 扁平层级 → 引入对比（更大的标题、更显眼的 primary CTA、一致中性的正文）
- 反向层级 → 交换信号（压低吵闹的不重要元素，重新定位被埋没的 primary）
- 单调 → 引入一次策略性打破；混乱 → 收敛到最强的模式

对于模棱两可的情况，倾向于更激进的层级——太强容易调弱，太弱则难以调强。

总结：按类别分类发现并修复的问题、用户应复审的判断取舍、开放建议。
