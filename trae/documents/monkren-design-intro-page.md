# Monkren Design 介绍网页实施计划

## 目标

参照现有 `index.html`（Visual Taste Lab）的网页框架和视觉品质，为 monkren-design 创建一个专业的介绍网页，展示其设计审查专家的核心能力。

## 设计方向

### 视觉身份

- **品牌定位**：设计审查专家——专业、精确、有态度
- **色调**：采用深墨色（ink）+ 暖纸色（paper）+ 赤陶强调色（terracotta）的克制配色，呼应"审查"的严肃性与"设计"的审美力
- **排版**：serif 标题 + sans 正文 + mono 代码，与现有页面保持同等品质
- **风格流派**：参考项目自有的 01 Pentagram 信息建筑风格——理性、克制、网格，符合"审查专家"的身份

## 页面结构（7 个区块）

### 1. 顶部导航（Header）

- 品牌：圆形标记 "M" + "Monkren Design"
- 导航链接：能力 / 维度 / 工作流 / 使用

### 2. Hero 区块

- **Eyebrow**：Design Review Expert
- **主标题**：提交设计，拿回一份专业审查报告。
- **副标题**：3 分钟，5 维度评审 + 硬编码检测 + 设计系统合规检查。不是泛泛而谈的「看起来不错」——是每个问题都带具体数值和修复建议的专业报告。
- **CTA 按钮**：查看能力 / 安装 Skill
- **右侧面板**：模拟审查报告预览（SVG 雷达图 + 5 维度评分条 + 严重度标记）

### 3. 能力总览区块（What it does）

- **Eyebrow**：核心能力
- **标题**：一个 Skill，覆盖设计审查全链路。
- **4 张能力卡片**：
  1. 5 维度评审体系（哲学一致性/视觉层级/细节执行/功能性/创新性）
  2. 硬编码值检测（颜色/字体/间距硬编码扫描 + 替换建议）
  3. 设计系统合规检查（组件使用/品牌资产/token 引用合规）
  4. 反 AI slop 检查（紫渐变/emoji/圆角 border accent 等 slop 清单）

### 4. 5 维度详解区块（5 Dimensions）

- **Eyebrow**：5 维度评审
- **标题**：每个维度独立评分，薄弱处一目了然。
- **3 个演示面板**（参照原页面的 demo-stack 结构）：
  - **面板 1：SVG 雷达图可视化** — 展示 5 轴雷达图 + 评分分布
  - **面板 2：评分纪律** — 4 条铁律（禁止通胀/禁止上浮/必须引证/创新性允许低分）
  - **面板 3：问题严重度分级** — P0⚠️致命 / P1⚡重要 / P2💡优化

### 5. 工作流区块（How it works）

- **Eyebrow**：审查工作流
- **标题**：从提交到报告，7 步完成专业审查。
- **7 个步骤**（参照原页面的 steps 结构）：
  1. 审查前发现 — 确认输入格式/设计类型/审查范围
  2. 理解审查对象 — 截图/HTML/代码/Figma？整体还是局部？
  3. 提取设计上下文 — 找到参照系（设计系统/品牌规范）
  4. 品牌资产提取 — 5 步系统化提取品牌资产
  5. 执行审查 — 5 维度 + 硬编码检测 + 合规性检查
  6. 输出报告 — SVG 雷达图 + 评分 + Keep/Fix/Quick Wins
  7. 自检迭代 — 确保报告无漏洞才交付

### 6. 使用区块（Use the Skill）

- **Eyebrow**：装上就能用
- **标题**：在你的 agent 里对话即审查。
- **左侧终端面板**：安装命令 + 使用示例
  ```
  npx skills add monkren/monkren-design
  ```
  使用示例对话：
  - 「帮我审查这个设计，5 维度打分」
  - 「检测这段代码里的硬编码颜色和间距值」
  - 「这个页面有没有违反设计系统规范？」
- **右侧卡片**：跨 agent 通用说明
  - Claude Code / Cursor / Codex / OpenClaw / Hermes 都能装
  - 没有按钮、没有面板。对话即审查。

### 7. 页脚（Footer）

- 版权信息 + License 说明（个人使用免费，企业商用需授权）
- GitHub 链接

## 技术实现

### 文件修改

- **修改** `/Users/monkren/Documents/GitHub/monkren-design/index.html`：用新的 monkren-design 内容替换现有 Visual Taste Lab 内容

### 实现要点

1. **保持相同的 CSS 架构**：复用 `:root` 变量系统、`.wrap` 容器、响应式断点
2. **保持相同的 HTML 语义结构**：`header.top` → `main` → `section` → `footer`
3. **SVG 雷达图**：内联 SVG 实现 5 轴雷达图，与审查报告中的雷达图规范一致
4. **审查报告预览面板**：模拟真实审查报告的视觉样式
5. **评分条可视化**：5 维度用进度条 + 分数展示
6. **严重度标记**：P0⚠️ / P1⚡ / P2💡 使用不同颜色
7. **响应式**：保持与原页面相同的 3 个断点（920px / 560px）
8. **无外部依赖**：纯 HTML + CSS，无 JS 框架，无外部字体/图片
