# 优化审查报告与评分提升建议文档 Spec

## Why

当前审查报告存在两个核心缺陷：
1. **报告仅输出在对话中，无法持久化**——审查报告作为对话文本输出，用户无法保存、归档、版本对比或分享。审查结果是一次性的，无法形成改进追踪闭环。
2. **缺少评分提升建议文档**——审查报告告诉用户"哪里有问题"（诊断），但缺少一份独立的、可操作的"如何提升评分"文档。用户拿到 6.6/10 的评分后，需要一份聚焦于"从 6.6 提升到 8.5 具体做什么"的行动指南，而非在冗长的审查报告中翻找。

优化目标：审查报告和评分提升建议文档均保存为 .md 格式文件，用户可归档、版本对比、分享给团队。

## What Changes

- **新增报告持久化机制**：审查报告完成后，自动保存为 `.monkren/reviews/{project}-{date}/review-report.md` 文件
- **新增评分提升建议文档**：审查完成后，额外生成一份独立的评分提升建议文档 `.monkren/reviews/{project}-{date}/score-improvement.md`，聚焦于每个维度的评分提升路径
- **更新 SKILL.md**：在执行层新增报告持久化规则和评分提升建议文档模板
- **更新 references/implementation.md**：补充报告持久化流程和评分提升建议文档生成规则

## Impact

- Affected specs: SKILL.md（执行层/报告持久化 + 评分提升建议模板）、references/implementation.md（报告持久化流程 + 评分提升建议文档生成规则）
- Affected code: 无代码文件变更，全部为 Markdown 文档更新

## ADDED Requirements

### Requirement: 审查报告持久化

审查报告完成后 SHALL 自动保存为 .md 格式文件，而非仅输出在对话中。

#### Scenario: 完整报告保存
- **WHEN** 审查完成并输出完整报告（三层递进结构）
- **THEN** 报告自动保存为 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/review-report.md`，包含完整的结论层 + 诊断层 + 行动层

#### Scenario: 快速报告保存
- **WHEN** 审查完成并输出快速报告
- **THEN** 报告自动保存为 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/review-report.md`，包含结论层 + 行动层

#### Scenario: 设计建议报告保存
- **WHEN** 输出设计建议报告
- **THEN** 报告自动保存为 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/design-suggestion.md`

#### Scenario: 保存后通知用户
- **WHEN** 报告保存完成
- **THEN** 在对话中告知用户文件保存路径，并提供简短摘要

#### Scenario: 同一项目多次审查
- **WHEN** 同一项目在不同日期进行多次审查
- **THEN** 每次审查保存到独立日期目录，支持版本对比

#### Scenario: 参考截图保存
- **WHEN** 审查过程中获取了参考截图
- **THEN** 截图保存到 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/references/` 目录，报告中的图片引用使用相对路径

### Requirement: 评分提升建议文档

审查完成后 SHALL 额外生成一份独立的评分提升建议文档，聚焦于每个维度的评分提升路径。

#### Scenario: 评分提升建议文档结构
- **WHEN** 生成评分提升建议文档
- **THEN** 文档包含：当前评分总览 + 各维度提升路径 + 优先级排序的提升路线图 + 预期评分提升效果

#### Scenario: 各维度提升路径
- **WHEN** 某维度评分低于 8 分
- **THEN** 为该维度提供：当前评分 + 目标评分 + 从当前到目标的具体步骤（按优先级排序）+ 每步的预期评分提升 + Quick Win

#### Scenario: 维度评分已达 8+ 时
- **WHEN** 某维度评分已达 8 分或以上
- **THEN** 该维度仅提供"维持建议"（1-2 条保持当前水平的注意事项），不生成提升路径

#### Scenario: 提升路线图
- **WHEN** 生成评分提升建议文档
- **THEN** 文档末尾包含按优先级排序的提升路线图，标注：步骤序号 / 涉及维度 / 具体操作 / 预估工作量 / 预期评分提升 / 依赖关系

#### Scenario: 预期评分提升效果
- **WHEN** 某个提升步骤完成
- **THEN** 标注预期评分提升效果，如"完成此步后，细节执行预计从 5/10 提升至 6.5/10"

#### Scenario: 与审查报告的关系
- **WHEN** 评分提升建议文档引用审查报告中的问题
- **THEN** 使用相对引用格式，如"详见 review-report.md 行动层 Fix #2"

### Requirement: 评分提升建议文档模板

评分提升建议文档 SHALL 使用以下模板结构：

```

| 维度 | 当前 | 目标 | 差距 | 提升难度 |
```

## MODIFIED Requirements

### Requirement: 审查工作流 Step 5 输出报告

审查工作流 Step 5 从"输出报告"扩展为"输出报告 + 持久化 + 生成评分提升建议"。

旧流程：
1. 输出报告到对话

新流程：
1. 输出报告到对话
2. 保存报告为 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/review-report.md`
3. 生成评分提升建议文档并保存为 `.monkren/reviews/{project-slug}-{YYYY-MM-DD}/score-improvement.md`
4. 告知用户文件保存路径

### Requirement: 审查工作流总结规则

总结规则新增文件保存路径提示。

旧格式：
```
✅ 审查完成。总体评分 X.X/10（评级）。
主要问题：[P0 问题数] 处阻断、[P1 问题数] 处重要、硬编码值 N 处。
关键 Quick Win：[最关键的 1 条] → 预计 5 分钟。
详见完整报告（结论层 → 诊断层 → 行动层）。
```

新格式：
```
✅ 审查完成。总体评分 X.X/10（评级）。
主要问题：[P0 问题数] 处阻断、[P1 问题数] 处重要、硬编码值 N 处。
关键 Quick Win：[最关键的 1 条] → 预计 5 分钟。

📄 报告已保存：.monkren/reviews/{project}-{date}/review-report.md
📊 评分提升建议：.monkren/reviews/{project}-{date}/score-improvement.md
```
