# 设计方向顾问全链路优化 Spec

## Why

monkren-design 技能的"设计方向顾问"能力(用户在模糊需求时获得 3 个差异化设计方向建议)是核心价值之一,但当前实现存在 4 个系统性问题:

1. **触发条件粗糙** — 仅 3 条模糊规则,无法区分"冷启动""增量设计""方向诊断""瓶颈突破""标杆对齐"5 类场景,导致输出深度不匹配
2. **底层数据不足** — 40 哲学库 + 兼容性矩阵 + 反模式齐备,但**缺乏项目类型→流派映射**(无法快速过滤)、**缺乏真实产品案例**(铁律 3 Evidence 必填无依据)、**缺乏约束过滤维度**(团队/技术栈/预算等无评分)
3. **输出结构松散** — 50-100 字 why + 3-4 特征 + 3-5 关键词,字段定义不严格,易产生空话、无依据的推荐
4. **质量保证薄弱** — 仅 1 条硬规则(3 方向必须跨流派),无空话黑名单、无 3 问适配性自检、无 evidence 必填、无互斥自检

本次升级通过**全链路重构**(哲学库 + 顾问规则 + 报告模板),把"主观建议"转化为"可验证、有依据、可对比的决策辅助"。

## ADDED Requirements

### Requirement: 5 类触发场景识别

agent 必须从 5 类场景(冷启动/增量设计/方向诊断/瓶颈突破/标杆对齐)中识别用户需求,据此调整输出重点。

#### Scenario: 冷启动
- **WHEN** 用户信号含"新项目""没 design system""从零开始"
- **THEN** agent 强调入门成本、组件库适配、第一个可落地方向
- **SHALL** 询问式确认用户场景阶段

#### Scenario: 增量设计
- **WHEN** 用户信号含"已有 system""新增模块/页面"
- **THEN** agent 强调风格延展性、token 复用、组件一致性

#### Scenario: 方向诊断
- **WHEN** 用户信号含"已有设计""想识别流派"
- **THEN** agent 强调归类、特征对比、改进方向

#### Scenario: 瓶颈突破
- **WHEN** 用户信号含"陷同质化""想换风格"
- **THEN** agent 强调差异最大化、风险标注、迁移路径

#### Scenario: 标杆对齐
- **WHEN** 用户信号含"对标 XX 产品/品牌"
- **THEN** agent 强调路径、改造幅度、可行性、所需资源

### Requirement: 6 字段固定输出结构

每个方向必须包含 6 个字段,缺一不可:1)流派+哲学 2)为什么适合 3)视觉 DNA 4)产品案例 5)适配维度评分 6)风险与反模式。

### Requirement: 5 铁律质量保证

每份设计建议报告必须通过 5 铁律自检:1)空话黑名单 2)3 问适配性自检 3)Evidence 必填 4)互斥自检 5)不堆叠无依据特征。

### Requirement: 项目类型→流派适配矩阵

philosophy.md §5 必须包含 7 项目类型(B2B SaaS/内容媒体/创意工具/电商/移动 App/公共服务/品牌官网)× 10 流派 = 70 个适配度评分。

### Requirement: 真实产品案例库

philosophy.md §6 必须包含 40 哲学 × 2-3 真实产品案例(总计 ≥ 80 个),作为铁律 3 Evidence 必填的数据源。

### Requirement: 约束过滤维度

philosophy.md §7 必须包含 7 维度(团队规模/技术栈/预算/国际化/暗色模式/维护周期/目标用户)× 10 流派 = 70 个评分。

### Requirement: 3 方向横评对比表

3 个方向输出后必须给一张对比表(7 维度:流派/视觉强度/团队适配/实施成本/风险等级/改造幅度/最适合)。

### Requirement: 选完后 3 个动作

报告末尾必须给 3 个可执行动作(5 分钟/30 分钟/1 天)。
