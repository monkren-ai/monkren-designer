## 一、项目总览

monkren-design 是一个专业设计审查 Skill，采用三层架构组织内容：

| 层级 | 根本问题 | 文件 |
|------|---------|------|
| 信念层 | 我信什么？ | SKILL.md §0-4 + philosophy.md |
| 标准层 | 设计必须满足什么标准？ | SKILL.md §2.1-2.13 + aesthetics.md + design-system.md + platform-guides.md + review-perspectives.md + lazyweb-integration.md |
| 执行层 | 我怎么做审查？ | SKILL.md §1-4 + implementation.md + lazyweb-integration.md |

**内容规模**：
- 主文档 SKILL.md：550+ 行
- 7 个 references 文件：共约 4000+ 行
- 7 个子 skill：各自独立的 SKILL.md

## 四、子 Skill 功能定位索引

| 子 Skill | 核心动作 | 输入 | 输出 | 依赖 Lazyweb | 依赖 browse | 设计流程位置 |
|----------|---------|------|------|:---:|:---:|---------|
| visual-taste-lab | 定义视觉身份 | 截图/URL/品牌参考 | design-language.md + VI brief | 否 | 否 | 设计前（策略层） |
| design-research | 深度调研 | 设计问题/界面类型 | 结构化研究报告 | 是 | 是 | 设计前（调研层） |
| design-brainstorm | 跨域灵感 | 设计问题/品类 | 创意报告 + ASCII 线框 | 是 | 是 | 设计前（灵感层） |
| quick-references | 速查参考 | 界面类型关键词 | 截图分组文档 | 是 | 是 | 设计中（参考层） |
| design-improve | 对比改进 | 现有设计截图 | 改进建议 + 参考截图 | 是 | 是 | 设计后（改进层） |
| add-inspo-source | 连接灵感源 | 灵感库名称/URL | libraries.json 配置 | 否 | 是 | 基础设施（配置层） |
| remove-inspo-source | 断开灵感源 | 灵感库名称 | 更新后的配置 | 否 | 否 | 基础设施（配置层） |

### 设计流程典型链路

```
add-inspo-source（配置数据源）
  → visual-taste-lab（定义视觉身份）
  → design-research / design-brainstorm（调研 + 灵感）
  → 用户实现设计
  → design-improve（对比改进）
  → 主 skill monkren-design（专业审查）
  → quick-references（随时速查参考）
```

### 子 Skill 边界速查

| 场景 | 应使用 | 原因 |
|------|--------|------|
| 还没设计，想了解领域现状 | design-research | 同品类深度研究，找最佳实践 |
| 还没设计，想要创意灵感 | design-brainstorm | 跨品类广度探索，找可迁移模式 |
| 已有设计，想改进 | design-improve | 找相似参考 + 具体改进建议 |
| 只想快速看几个参考截图 | quick-references | 轻量速查，不做深度分析 |
| 想建立品牌视觉系统 | visual-taste-lab | 从品牌身份出发的系统化视觉方向 |
| 想连接/断开灵感库 | add/remove-inspo-source | 配置管理 |
